## Instructors: where your lesson goes

1. Make a folder `lessons/NN-your-slug/` (copy
   [`01-github-and-claudelab/`](01-github-and-claudelab/) as a structural
   template).
2. Write a participant-facing `README.md` in it.
3. Add your row to the schedule above, and to the table in the top-level
   [`README.md`](../README.md).

See [`../instructors/README.md`](../instructors/README.md) for the writing
standard and workflow.

## Threads that run through the workshop

From the event vision — the everyday work of scientific research, with Claude
writing the code and the participant learning to explain what they want:

- **The request → review → validate loop** — direct Claude, review what it did,
  confirm by result.
- **Sharing** — GitHub (primary), vs. Drive / OneDrive / iCloud.
- **Building a knowledge base** — docs, todos, then skills, commands, scripts —
  kept in your ClaudeLab.
- **Design** — HTML + JavaScript; clear visual descriptions; quick local GUI
  tools; vector figures.
- **Data** — cleanup, plots with Python, small Python tools.
- **Extending** — a simple MCP server in Python.

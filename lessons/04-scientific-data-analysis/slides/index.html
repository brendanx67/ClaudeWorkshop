Before the science

Set up, and two habits
that pay off all day

The unglamorous part that makes everything after it easy.

Set up the project

  • Copy data/ and the example CLAUDE.example.md into a new folder in your ClaudeLab — then rename it to CLAUDE.md
  • Open a new Claude Code session in that folder
  • Have Claude scaffold the project — setup only, no analysis yet If Python's missing it installs it with uv, creates the venv, installs packages.
  • Open the lightweight CLAUDE.md together — walk the decisions and rules it carries

Two habits that make it work

Write it down, once

  • Save everything to Markdown files
  • Point CLAUDE.md at them — don't restate This is how you start a new session cold and pick up where you stopped.

Findings files

  • Have Claude save findings as it goes, named well
  • Self-contained: status, discussion, figures, tables, lists
  • Full provenance — the data & scripts behind each one A finding you can't retrace is a rumor.

The process

The analysis, step by step

The order matters. The beginning carries everything.

Step 1 · The science

Let Claude interview you

  • Domain, the experiment, hypotheses & questions, the goal of the study
  • One question at a time — no walls of questions
  • Save it to PROJECT.md, so it carries into future sessions You can't analyze what you can't state.

Step 2 · The metadata

Know every column before you trust it

  • Have Claude report the columns and the value types in each
  • Let it ask questions; correct its mistakes — you're the expert
  • Visualize the key variables (Lesson 3, yesterday)
  • Mark which samples are controls, which are experimental
  • Save the metadata findings to METADATA.md, and point CLAUDE.md at it

Step 3 · The data

What are you actually looking at?

  • What's the shape — are rows samples, or proteins?
  • What kind of values are stored? Raw, or already transformed?
  • How bad is the missingness?
  • Save the data findings to DATA.md, and point CLAUDE.md at it

Step 4 · Data loaders

Real work begins

  • Testing & validating this code is essential — every downstream script uses it
  • Decide now: normalization, batch correction, how to handle missingness
  • Discuss it through, then have Claude build the reusable loader — tested, linted, typed

Step 5 · Quality control

Look before you leap

  • CV plots — on linear-scale data, not logged — especially among controls
  • PCA plots colored by metadata variables
  • How does normalization change CV and PCA — especially in the controls?
  • Abundance distributions across runs; correlation plots
  • Save the good ones as findings — and pull them into an HTML QC report One page you can scroll and share; a preview of the final report.

Step 6 · Exploration & analysis

Now, the real questions

  • Univariate: proteins vs. exposure, dose, dose rate — plots & lists, saved as findings
  • Predict dose from the proteome — regularized regression; let Claude talk you through it
  • GO (Gene Ontology) enrichment of the significant proteins
  • Can Claude build a story from the hits? This is where most of the two hours goes.

Step 7 · The report

Assemble it from your findings

  • Claude builds a report of findings & methods from the saved findings files
  • Output as HTML or Markdown (→ PDF)
  • Something you'd hand a collaborator — or your future self

Push everything to GitHub

  • It backs up your work
  • It never overwrites earlier sessions' findings
  • It lets you pick up later, from anywhere This is why Day 1 was about Git.

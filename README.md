# Claude Code for Scientific Researchers

Materials for a hands-on workshop on using **Claude Code** in everyday scientific
research — exploring and cleaning data, building analyses and figures, and
keeping a durable, shareable record of your work.

Built and taught by a team from **UW Genome Sciences — MacCoss lab**, including
researchers and developers from the [Skyline](https://skyline.ms) project. Shared
openly: other Claude Community Ambassadors are welcome to read, borrow, and
adapt anything here.

> **You bring:** a Mac or Windows laptop, a paid Claude plan (Pro or Max), the
> [Claude Desktop app](https://claude.com/download), and a free
> [GitHub account](https://github.com/signup).
> **You leave:** able to use Claude in your own research — not just having
> watched someone else do it.

## Two repositories: a textbook and a lab

The workshop uses two repositories that sit side by side:

- **ClaudeWorkshop** (this repo) — the **textbook**. Lessons, setup instructions,
  and instructor notes. Shared by everyone; you clone it to read along.
- **ClaudeLab** — your **lab**. A repository *you* create and own, on *your*
  GitHub, where you run experiments with Claude and keep the results. You make it
  in the first lesson.

You'll open Claude Code once, on a single folder that holds both — a
"multi-repo root" (details in [`setup/`](setup/)). The habit that unlocks: adding
another project later is just *"clone it into this folder."*

## The two mornings

Both run 8:00 AM – 12:00 PM (July 21–22). Arrival & setup at 8:00, hands-on at
8:30, coffee at 10:00, second session at 10:30, wrap by noon. Full schedule and
links: [`lessons/README.md`](lessons/README.md).

| Day | Session | Topic |
|-----|---------|-------|
| **1** | Setup | GitHub and your ClaudeLab |
| **1** | Design | Design with HTML and JavaScript |
| **1** | Data | Data cleanup and visualization |
| **2** | Analysis | Scientific data analysis |
| **2** | Notebooks | ELN reviews |
| **2** | Extending | Extending Claude Code with MCP servers (Python) |

## What's here

- [`setup/`](setup/) — how a participant gets from "Claude Desktop installed" to
  ready for lesson one.
- [`lessons/`](lessons/) — one folder per session; each is a participant guide.
- [`instructors/`](instructors/) — the teacher's-edition back matter: open notes
  on how we're building and teaching this, plus a UW-branded HTML
  presentation template. For the team and any curious Ambassador; participants can
  ignore it.
